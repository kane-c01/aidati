## 一、环境矩阵

| 环境 | 域名 | 部署目标 | 数据库 | 是否启用监控 |
|---|---|---|---|---|
| dev(本地) | localhost | Docker Compose | 本地 MySQL/Redis | 否 |
| test | api-test.yourdomain.com | 单台 ECS / Docker | 测试 RDS | 基础 |
| staging | api-staging.yourdomain.com | 与生产同结构,缩容 | 独立 RDS | 完整 |
| prod | api.yourdomain.com | 生产 | 主从 RDS | 完整 |

> **强制**:test/staging/prod 全部使用 HTTPS,生产证书 1 年有效;到期前 30 天自动告警(运维平台或 acme.sh)

## 二、本地开发环境

### 2.1 必备依赖

```
Node.js >= 20 LTS
Python >= 3.11
pnpm >= 9
Docker >= 24
微信开发者工具(稳定版)
```

### 2.2 一键启动

`deploy/docker-compose.yml`(节选):

```yaml
version: '3.9'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: dev_root
      MYSQL_DATABASE: ai_quiz
      MYSQL_USER: app
      MYSQL_PASSWORD: dev_app
    ports: ["3306:3306"]
    volumes: ["./data/mysql:/var/lib/mysql"]

  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]

  minio:
    image: minio/minio
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
    ports: ["9000:9000", "9001:9001"]

  backend:
    build: ../backend
    env_file: ../backend/.env.dev
    depends_on: [mysql, redis]
    ports: ["3000:3000"]

  ai-service:
    build: ../ai-service
    env_file: ../ai-service/.env.dev
    depends_on: [redis]
    ports: ["8000:8000"]

  admin-web:
    build: ../admin-web
    ports: ["5173:5173"]
```

启动:`docker compose -f deploy/docker-compose.yml up -d`

## 三、构建与发布

### 3.1 后端 Docker 镜像

`backend/Dockerfile`:

```dockerfile
FROM node:20-alpine AS deps
WORKDIR /app
COPY package*.json pnpm-lock.yaml ./
RUN corepack enable && pnpm install --frozen-lockfile

FROM node:20-alpine AS builder
WORKDIR /app
COPY . .
COPY --from=deps /app/node_modules ./node_modules
RUN pnpm build

FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY package.json ./
EXPOSE 3000
CMD ["node", "dist/main.js"]
```

### 3.2 AI 服务 Docker

`ai-service/Dockerfile`:

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 8000
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000", "--workers", "2"]
```

### 3.3 CI(GitHub Actions 示例)

`.github/workflows/ci.yml`:

```yaml
name: CI
on: [push, pull_request]

jobs:
  backend-test:
    runs-on: ubuntu-latest
    services:
      mysql:
        image: mysql:8.0
        env:
          MYSQL_ROOT_PASSWORD: ci
          MYSQL_DATABASE: ai_quiz_test
        ports: ["3306:3306"]
      redis:
        image: redis:7
        ports: ["6379:6379"]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20 }
      - run: corepack enable
      - working-directory: backend
        run: |
          pnpm install --frozen-lockfile
          pnpm lint
          pnpm test --coverage
          pnpm build

  ai-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: 3.11 }
      - working-directory: ai-service
        run: |
          pip install -r requirements.txt
          pip install pytest
          pytest -q

  miniprogram-build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - working-directory: miniprogram
        run: |
          pnpm install
          pnpm lint
          pnpm tsc --noEmit
```

### 3.4 CD(简化方案)

- **后端 / AI 服务**:推 main → 构建镜像 → 推 ACR / TCR → ssh 至生产 → `docker compose pull && up -d`(蓝绿部署:`-blue` / `-green` 双副本,Nginx 切流)
- **管理后台**:推 main → 构建静态资源 → 上传 OSS → 刷新 CDN
- **小程序**:不走 CI 自动发布(微信工具只能本地或 miniprogram-ci),由人手在「微信开发者工具」上传体验版,审核后推送

`miniprogram-ci` 可选自动化:

```bash
miniprogram-ci upload \
  --pp ./miniprogram \
  --pkp ./private.key \
  --appid wx1234567890 \
  --uv 1.0.0 \
  --ud "$COMMIT_SHA"
```

## 四、Nginx 配置示例

`deploy/nginx/api.conf`:

```nginx
server {
    listen 443 ssl http2;
    server_name api.yourdomain.com;

    ssl_certificate /etc/ssl/api.yourdomain.com.crt;
    ssl_certificate_key /etc/ssl/api.yourdomain.com.key;
    ssl_protocols TLSv1.2 TLSv1.3;

    client_max_body_size 50M;

    # 限流(每个 IP 100/min)
    limit_req_zone $binary_remote_addr zone=apilim:10m rate=100r/m;
    limit_req zone=apilim burst=20 nodelay;

    # 日志结构化
    log_format json_combined escape=json '{"time":"$time_iso8601","ip":"$remote_addr",'
        '"req":"$request","status":$status,"bytes":$body_bytes_sent,'
        '"rt":$request_time,"ua":"$http_user_agent","rid":"$http_x_request_id"}';
    access_log /var/log/nginx/api.access.log json_combined;
    error_log  /var/log/nginx/api.error.log warn;

    # 健康检查直通
    location = /healthz { proxy_pass http://backend:3000; }

    # 业务 API(NestJS)
    location /v1/ {
        proxy_pass http://backend:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Request-Id $http_x_request_id;
        proxy_read_timeout 90s;
    }

    # AI 编排(内部访问,生产不暴露公网,仅供 backend 调用)
}

server {
    listen 80;
    server_name api.yourdomain.com;
    return 301 https://$host$request_uri;
}
```

## 五、密钥与凭证管理

### 5.1 原则

- **任何密钥不进 git**:加 `.env`、`*.pem` 到 `.gitignore`,启用 git-secrets / gitleaks 扫描
- **生产密钥**只在生产环境变量 / 阿里云 KMS / 腾讯云 SSM 中配置,运维 SSO 二次校验
- **本地开发**用各服务的 dev/test key,绝不混用生产
- **轮换周期**:JWT secret 每 90 天轮换;LLM API key 季度复核

### 5.2 密钥清单

| 名称 | 来源 | 存储位置 | 谁有权限 |
|---|---|---|---|
| WECHAT_APPID/SECRET | 小程序后台 | 生产环境变量 | DevOps 主负责人 |
| JWT_SECRET / REFRESH_SECRET | 自生成(`openssl rand -hex 32`) | 同上 | 同上 |
| OSS Access Key / Secret | 云控制台,**仅 STS 临时 KEY 在前端** | 后端环境变量 | DevOps 主负责人 |
| LLM API Keys (DeepSeek/Qwen/GLM) | 各 LLM 控制台 | 同上 | 同上 |
| Sentry DSN | Sentry 后台 | 同上(写入级别) | 同上 |
| 数据库连接串(prod) | RDS 控制台 | 同上 + 限制 IP 白名单 | 同上 |

### 5.3 微信内容安全 API 调用

后端使用 `access_token`(**全局唯一**),需要:
- 单独服务 / 定时器维护(7200s 刷新,提前 5 分钟续期)
- Redis key:`wx:access_token`,带 expireAt
- **同一 appid 全集群共享**,避免重复请求被微信限流(每日 2000 次)

## 六、监控与日志

### 6.1 应用日志

- **统一格式**(JSON Lines):
  ```jsonc
  {"ts":"2026-05-05T03:14:00Z","level":"info","service":"backend","msg":"paper.generate.start","user_id":"1001","paper_id":"2002","request_id":"...","trace":[]}
  ```
- 通过 stdout 输出,Docker 日志驱动收集到阿里云 SLS / Loki
- 敏感字段(token、openid)写入前 mask:`14****56`

### 6.2 关键日志事件(必埋)

| 事件 | 字段 |
|---|---|
| `auth.login` | user_id, role, is_first |
| `paper.generate.start/success/fail` | user_id, paper_id, model, tokens_in, tokens_out, ms |
| `paper.submit.success/fail` | user_id, paper_id, score, ms |
| `ocr.start/success/fail` | user_id, photo_set_id, pages, ms |
| `moderation.block` | user_id, scene, reason |
| `quota.exceeded` | user_id |
| `admin.action` | admin_id, action, target_type, target_id |

### 6.3 指标(Prometheus)

| 指标 | 类型 | 说明 |
|---|---|---|
| `http_request_duration_seconds` | histogram | API 耗时,labels: method/route/status |
| `http_request_total` | counter | API 调用数 |
| `paper_generate_duration_seconds` | histogram | 出题任务耗时 |
| `paper_generate_total` | counter | 任务数,labels: status/model |
| `llm_tokens_total` | counter | tokens 用量,labels: provider/model |
| `llm_cost_yuan_total` | counter | LLM 成本(估算) |
| `queue_depth` | gauge | 队列堆积深度 |
| `db_connections` | gauge | DB 连接池 |

### 6.4 告警(关键)

| 触发条件 | 通知方式 | 责任人 |
|---|---|---|
| API 5xx 错误率 > 1%(5min) | 飞书 / 企微 webhook | 后端值班 |
| 出题成功率 < 90%(15min) | 同上 | 后端值班 |
| LLM 主备都失败 | 同上 + 短信 | 后端值班 |
| MySQL 连接 > 80% 上限 | 同上 | DBA |
| Redis 内存 > 80% | 同上 | DevOps |
| 磁盘 > 80% | 同上 | DevOps |
| HTTPS 证书 < 30 天 | 邮件 / 飞书 | DevOps |
| 月度 LLM 成本 > 预算 80% | 同上 | 产品 + 运维 |

### 6.5 SLO

| 指标 | SLO |
|---|---|
| API 可用性 | 99.5%/月(MVP),99.9%(成熟期) |
| API P95 延迟 | < 500ms |

## 七、安全清单

### 7.1 网络层

- [ ] 全站 HTTPS,弱协议禁用(SSL3、TLS1.0/1.1)
- [ ] HSTS 头(`max-age=31536000; includeSubDomains; preload`)
- [ ] WAF(腾讯云/阿里云):基础规则(SQL 注入、XSS、爬虫识别)
- [ ] 内网与公网隔离:数据库 / Redis 只允许后端 SG 访问
- [ ] SSH 仅密钥登录,关闭密码;堡垒机 + 二次验证

### 7.2 应用层

- [ ] 所有输入参数使用 zod / class-validator 校验
- [ ] Prisma / TypeORM 防 SQL 注入(始终参数化查询)
- [ ] OpenID 哈希存储(SHA-256 + 盐)用于审计;原值仅用于授权
- [ ] JWT secret 高熵,过期 + refresh
- [ ] CORS 白名单(管理后台域名)
- [ ] 文件上传 MIME 类型与扩展名双重校验,**严禁**接受 `.exe/.html/.svg`(SVG 含脚本)
- [ ] PDF 上传服务端再校验首部 magic number,阻止伪造
- [ ] `Content-Security-Policy`、`X-Content-Type-Options: nosniff` 等安全头
- [ ] 每次 release 前跑 `npm audit` / `pip-audit`,高危漏洞必修

### 7.3 数据层

- [ ] 用户敏感字段(若 V3 加支付)加密存储(AES-256-GCM)
- [ ] 注销冷静期 7 天,期满**确实**删除(脚本日志记录)
- [ ] PII 不进日志(token / openid / 手机号)
- [ ] DB 备份加密(KMS)
- [ ] 内容审核日志加密留存 ≥ 6 个月

### 7.4 第三方

- [ ] LLM:仅使用已备案模型;调用日志保留(prompt + response)≥ 6 个月
- [ ] 微信内容安全 API:全链路接入,故障降级到本地敏感词表
- [ ] OCR:主备双供应商;不接收图片回调

### 7.5 运维流程

- [ ] 生产任何变更都走 PR + 评审,严禁直登服务器改文件
- [ ] 上线时间窗:工作日 10:00–17:00,周末与晚间冻结(除 hotfix)
- [ ] Hotfix 流程:`hotfix/*` 分支 → 走 staging 验证 → 紧急合并 main
- [ ] 故障复盘机制:P0/P1 故障 7 日内完成 5why 复盘,产出改进项

### 7.6 应急响应

| 故障级别 | 触发 | 响应时长 | 处置流程 |
|---|---|---|---|
| P0 | 全站不可用 / 数据泄漏 | 15 分钟 | 立即拉群 + 短信值班长 + 启动应急预案 |
| P1 | 核心功能(出题/答题)不可用 | 30 分钟 | 拉群 + 通知 |
| P2 | 部分功能受影响 | 2 小时 | 工单跟踪 |
| P3 | 体验问题 | 1 工作日 | 排期修复 |
