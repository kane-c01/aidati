# AI 智能出题学习小程序 — UI 设计规范

> 版本:v1.0  
> 配套 PRD:v1.1  
> 适用对象:UI 设计师 / 小程序前端 / 评审  
> 最后更新:2026-05-05  
> 风格代号:**Quietly Smart**(安静而智能)

---

## 一、设计哲学

> 让用户**专注学习**,让工具**保持安静**。AI 不是炫技的明星,而是隐于幕后的好搭档。

### 1.1 三条核心原则

1. **少即是多**:每一屏只解决一件事;按钮、信息、装饰物默认更少。
2. **温柔克制**:学习是一件需要耐心的事,UI 不喧宾夺主、不大红大紫,色彩柔和、动效缓慢。
3. **诚实可靠**:AI 内容必须明确标识,错误必须给出明确下一步,不玩弄花招与营销话术。

### 1.2 风格关键词

`简洁` · `温暖` · `高效` · `克制` · `信任`

### 1.3 风格参考(气质对标,不照搬)

- Notion 的简洁排版
- Apple 的留白哲学
- 微信读书的中文阅读体感
- DeepSeek 的灰阶背景 + 紫色点缀

## 二、设计 Token(可直接落地为 CSS 变量)

### 2.1 颜色系统

#### 品牌色

| 名称 | 色值 | rgba | 用途 |
|---|---|---|---|
| **brand-primary** | `#5B5BD6` | rgba(91,91,214,1) | 主操作、强调、链接、Logo |
| brand-primary-hover | `#4949C4` | — | hover/active |
| brand-primary-disabled | `#C8C8F0` | — | 禁用 |
| brand-primary-bg | `#EEEEFC` | — | 主色填充背景(选中、激活) |
| brand-secondary | `#4A6FA5` | rgba(74,111,165,1) | 次操作、信息 chips |

#### 功能色

| 名称 | 色值 | 用途 |
|---|---|---|
| **success** | `#1DBF73` | 答对、审核通过、正向反馈 |
| success-bg | `#E5F8EE` | 提示背景 |
| **warning** | `#F5A623` | 警告、AI 置信度低、得分中等 |
| warning-bg | `#FEF3DD` | — |
| **danger** | `#E14C4C` | 答错、驳回、删除 |
| danger-bg | `#FCE8E8` | — |
| **info** | `#4A6FA5` | 一般信息提示 |

#### 中性色(文本与背景)

| 名称 | 色值 | 用途 |
|---|---|---|
| text-primary | `#1A1A1A` | 主标题、正文 |
| text-secondary | `#4A4A4A` | 次正文(白底偏暖) |
| text-tertiary | `#8C8C8C` | 辅助说明 |
| text-disabled | `#BFBFBF` | 禁用文字 |
| text-inverse | `#FFFFFF` | 反白(主色按钮上的字) |
| border | `#E8E6E1` | 卡片描边、分割线 |
| border-light | `#F0EEEA` | 弱描边 |
| **bg-page** | `#FAFAF7` | **页面底色(米白,护眼)** |
| bg-card | `#FFFFFF` | 卡片底色 |
| bg-elevated | `#FFFFFF` | 浮层底色 |
| bg-overlay | `rgba(20,20,20,0.50)` | 蒙层 |

> 与传统纯白 #FFF 不同,我们使用 `#FAFAF7` 米白,在长时间学习时降低视觉疲劳;卡片再叠白色形成柔和层级。

#### 暗色模式(V2)

| 名称 | 暗色 |
|---|---|
| bg-page | `#0F0F12` |
| bg-card | `#1A1A1F` |
| text-primary | `#F5F5F0` |
| brand-primary | `#7D7DE8`(略亮以保持对比) |

### 2.2 字号(rpx,设计稿宽 750)

| Token | rpx | px(@2x) | 字重 | 行高 | 用途 |
|---|---:|---:|---|---:|---|
| display | 60 | 30 | Bold(700) | 1.3 | 大标题(结果页分数) |
| h1 | 44 | 22 | Bold(700) | 1.4 | 页面主标题 |
| h2 | 36 | 18 | Medium(500) | 1.4 | 卡片标题 |
| h3 | 32 | 16 | Medium(500) | 1.5 | 次级标题 |
| body | 28 | 14 | Regular(400) | 1.6 | 正文 |
| body-strong | 28 | 14 | Medium(500) | 1.6 | 强调正文 |
| caption | 24 | 12 | Regular(400) | 1.5 | 辅助说明 |
| tiny | 22 | 11 | Regular(400) | 1.4 | 标签、徽标 |

字体栈:

```css
font-family:
  -apple-system, "PingFang SC", "Helvetica Neue",
  "Microsoft YaHei", "Source Han Sans CN", sans-serif;
```

> 不打包自定义字体,完全继承系统字体,降低包体积。

### 2.3 间距(8 栅格)

`spacing-xxs: 4 / xs: 8 / sm: 16 / md: 24 / lg: 32 / xl: 48 / xxl: 64`(单位 rpx)

> 90% 的间距用 `xs/sm/md/lg`。

### 2.4 圆角

| Token | rpx | 用途 |
|---|---:|---|
| radius-sm | 8 | 标签、Tag |
| radius-md | 16 | 卡片、输入框、按钮 |
| radius-lg | 24 | 大卡片、弹窗 |
| radius-pill | 999 | 胶囊按钮 |
| radius-full | 50% | 圆形头像 |

### 2.5 阴影

| Token | 值 | 用途 |
|---|---|---|
| shadow-card | `0 2rpx 12rpx rgba(0,0,0,0.04)` | 卡片默认 |
| shadow-card-hover | `0 4rpx 20rpx rgba(0,0,0,0.06)` | 按下/选中 |
| shadow-popover | `0 8rpx 32rpx rgba(0,0,0,0.10)` | 弹窗 |
| shadow-fab | `0 6rpx 16rpx rgba(91,91,214,0.30)` | 浮动按钮 |

> 阴影**克制**,不用过深的黑色,保持轻盈感。

### 2.6 动效

| Token | 时长 | 缓动 | 用途 |
|---|---|---|---|
| ease-quick | 150ms | `cubic-bezier(0.4, 0, 0.2, 1)` | 按下、悬停 |
| ease-base | 250ms | `cubic-bezier(0.4, 0, 0.2, 1)` | 默认转场 |
| ease-slow | 400ms | `cubic-bezier(0.2, 0.8, 0.2, 1)` | 弹窗、页面进入 |

> **永远不超过 400ms**,避免拖沓;**绝不弹跳超过 1 次**(过度活泼伤气质)。

### 2.7 z-index 层级

| Token | 值 |
|---|---:|
| z-base | 1 |
| z-sticky | 100 |
| z-fab | 500 |
| z-overlay | 1000 |
| z-modal | 1100 |
| z-toast | 1200 |
| z-loading | 1300 |

## 三、组件库

### 3.1 按钮(Button)

#### 类型

| 类型 | 视觉 |
|---|---|
| **primary** | 主色填充 + 白字 |
| secondary | 主色描边 + 主色字 + 白底 |
| ghost | 无边框 + 主色字(用于次要 link) |
| danger | 红色填充 + 白字(删除等危险操作) |
| disabled | 灰色填充 + 灰字 |

#### 尺寸

| 尺寸 | 高度 rpx | 字号 | padding 横 |
|---|---:|---|---:|
| large | 96 | 32 / Medium | 32 |
| medium | 80 | 28 / Medium | 24 |
| small | 56 | 24 / Regular | 16 |

按下态:scale(0.97) + 阴影减弱(150ms)

```
┌──────────────────────────────────┐
│            立即出题              │  ← primary, large, 全宽
└──────────────────────────────────┘
```

### 3.2 卡片(Card)

通用结构:

```
┌─[ 卡片,圆角 16rpx,阴影 shadow-card ]─┐
│ ┌──┐ 标题 (h2)                    >  │
│ │图│ 副标题/描述 (body, secondary)    │
│ └──┘                                 │
│ ──────────────────────────────────  │ ← border-light 分割
│ 标签 1   标签 2   标签 3              │
└──────────────────────────────────┘
```

变体:

- **书籍卡片**:左封面 + 右标题/作者/标签
- **错题卡片**:题干预览 + 错误次数徽标
- **试卷历史卡片**:标题 + 得分进度条 + 时间

### 3.3 输入框(Input / Textarea)

```
┌──────────────────────────────┐
│  请输入书名搜索...   [×]      │
└──────────────────────────────┘
   ↑ border 1rpx, 圆角 16, 高 88rpx
   focus: border 主色 + bg 白色
   error: border danger + 下方红色 caption
```

### 3.4 选择器(Picker / Selector)

#### 选项卡组(题型多选)

```
┌─────────┐  ┌─────────┐  ┌─────────┐
│  单选 ✓  │  │  判断   │  │  填空   │
└─────────┘  └─────────┘  └─────────┘
   ↑ 选中:主色描边 + 主色背景(brand-primary-bg)
   ↑ 未选:灰色描边 + 白底
```

#### 单选组(难度)

```
┌──────┬──────┬──────┐
│ 简单 │ 中等✓│ 困难 │   ← 分段控件,选中态主色填充
└──────┴──────┴──────┘
```

### 3.5 列表项(ListItem)

```
┌─────────────────────────────────────┐
│ ┌──┐ 主标题                          │
│ │图│ 副标题 (caption, secondary)     │
│ └──┘                              >  │
└─────────────────────────────────────┘
   ↑ 高度自适应,最小 144rpx,左右 padding 32
   ↑ 点击态:bg → border-light(150ms)
```

### 3.6 标签(Tag)

```
┌─ 已掌握 ─┐  ┌─ 中等难度 ─┐  ┌─ AI 生成 ─┐
└──────────┘  └────────────┘  └───────────┘
```

| 类型 | 样式 |
|---|---|
| 默认(灰) | 灰背景 + 二级文字 |
| 主色(强调) | 主色 bg(`brand-primary-bg`)+ 主色字 |
| 成功 | success-bg + success 字 |
| 警告 | warning-bg + warning 字 |
| 错误 | danger-bg + danger 字 |

圆角 `radius-sm`,字号 `tiny`,padding 4×12rpx。

### 3.7 Toast 与 Modal

#### Toast(轻反馈)

```
        ┌────────────────────┐
        │   ✓ 已加入错题本    │   ← 黑背景 80% 透明,白字,1.5s 自动消失
        └────────────────────┘
```

类型:`success` / `error` / `info` / `loading`(带菊花图标)

#### Modal(强确认)

```
┌──────────────────────────────────┐
│            标题(h2)               │
│                                  │
│  描述说明文字,1-2 行              │
│                                  │
│  ┌────────┐    ┌────────┐         │
│  │  取消  │    │  确定  │          │
│  └────────┘    └────────┘         │
└──────────────────────────────────┘
   ↑ 圆角 24rpx,白底,shadow-popover
   ↑ 蒙层 bg-overlay
```

### 3.8 空状态(EmptyState)

```
        🌱(简笔画 illustration)

         还没有错题哦~

      去做一套题试试看吧

     ┌─────────────────┐
     │     去出题       │     ← 仅在「需要引导动作」时出现
     └─────────────────┘
```

文案规则:
- 一句话主标题 + 一句解释 + (可选)CTA 按钮
- 永远不要写「无数据」「暂无内容」这种冷冰冰的话

### 3.9 骨架屏(Skeleton)

列表骨架:

```
┌─────────────────────────────┐
│ ▭▭▭▭▭▭▭▭▭▭▭▭▭▭▭▭▭▭▭        │  ← 浅灰矩形,1.5s 呼吸动效
│ ▭▭▭▭▭▭                     │
└─────────────────────────────┘
```

> 列表 / 详情页统一用骨架,**不用菊花转圈**。

### 3.10 进度条(Progress)

```
答题进度 ──────────────────────────  3/10
█████████░░░░░░░░░░░░░░░░░░░░░░  30%
```

- 顶部细线进度条:用于答题
- 圆形进度环:用于结果页主分数(承担情感重量)
- 出题加载页的「波浪式」分阶段进度(下文详述)

### 3.11 浮动操作按钮(FAB)

```
                    ┌─┐
                    │+│   ← 主色圆形,shadow-fab,悬浮于右下
                    └─┘
```

仅在「上传新书」(V2)等少数关键二级动作使用。

### 3.12 AI 内容标识(AI Disclaimer)— 全局必备组件

```
┌─ 🤖 内容由 AI 生成,仅供参考 ─┐
└─────────────────────────────┘
   ↑ 灰色 chip,左 robot 小图标(线条),tiny 字号,brand-secondary 字色
```

出现位置:
- 出题加载页 footer
- 答题页顶部 sticky(轻提示,不打扰)
- 结果页每道题展开时
- 所有 AI 解析、批改反馈周围

### 3.13 配额徽标(QuotaBadge)

```
┌─────────────────┐
│ 今日剩余 7/10 次  │   ← 米白底 + 主色描边,胶囊形
└─────────────────┘
```

放置在首页顶部、出题配置页右上,点击查看额度规则说明。

## 四、通用布局

### 4.1 导航栏(NavBar)

#### 样式

```
┌─[ 状态栏 ]────────────────────────┐
│ ←   页面标题             [右图标] │  ← 88rpx 高,白底,底部 1rpx 弱描边
└──────────────────────────────────┘
```

- 左侧返回箭头(自动隐藏 tabBar 页面)
- 标题居中,h2 字号
- 右侧最多 2 个图标(线条风格,touch 区域 88×88)

#### 沉浸式(结果页等)

```
┌── (透明渐变到内容色)──┐
│ ←                     │
└──────────────────────┘
```

### 4.2 TabBar(底部导航)

```
┌──────────────────────────────────┐
│    📚         📷         👤        │
│  书库       拍照       我的       │
└──────────────────────────────────┘
   ↑ 高 110rpx + 安全区,白底,顶部 1rpx 弱描边
   ↑ 选中:主色 + 加粗;未选中:三级文字
```

图标全部使用线条风格(选中变实心)。

### 4.3 安全区

- 顶部:状态栏 + NavBar 总高度
- 底部:`env(safe-area-inset-bottom)` 全局通过 `<safe-area-bottom>` 组件兜底

### 4.4 通用页面骨架

```
┌─────────────────────────────┐
│         NavBar              │  ← 固定
├─────────────────────────────┤
│                             │
│         内容区域(可滚动)      │
│         padding-x: md(24rpx)│
│                             │
├─────────────────────────────┤
│         TabBar(可选)        │  ← 固定
└─────────────────────────────┘
```

## 六、交互规范

### 6.1 手势

| 手势 | 行为 |
|---|---|
| 点击 | 主操作 |
| 长按 | 辅助菜单(如缩略图操作)或排序模式 |
| 左滑 | 列表项删除/标记掌握(可关) |
| 下拉 | 刷新 |
| 上拉 | 加载更多 |
| 双指缩放 | 仅图片预览支持 |

### 6.2 反馈层级

| 程度 | 用法 |
|---|---|
| 静默 | 内部状态变化(如选中) |
| Toast | 轻量,不阻塞(收藏成功/复制成功) |
| Inline 提示 | 输入框下方红色 caption(校验失败) |
| Modal | 强确认(删除/提交/退出) |
| 全屏页 | 出题失败、网络断开等致命 |

### 6.3 触觉反馈(振动)

- 答对:`wx.vibrateShort({ type: 'light' })`
- 答错 / 删除确认:`wx.vibrateShort({ type: 'medium' })`
- 强确认完成:`wx.vibrateLong()`(谨慎使用)
- 用户可在「设置」中关闭

### 6.4 加载策略

| 场景 | 方案 |
|---|---|
| 列表首次加载 | 骨架屏 |
| 列表分页加载 | 底部加载条 + 文字「加载中...」 |
| 提交动作 | 按钮内部 loading 图标 + 禁用 |
| 异步任务(出题/批改) | 专用页面或 sticky 进度条 |

### 6.5 错误状态

```
┌──────────────────────────────────┐
│                                  │
│         ⚠  插画(无信号)         │
│                                  │
│      网络似乎不太顺畅            │
│      请检查网络后重试            │
│                                  │
│      ┌────────────┐               │
│      │   重新加载  │              │
│      └────────────┘               │
└──────────────────────────────────┘
```

### 6.6 无障碍

- 所有可点击元素 touch 区 ≥ 88×88rpx
- 重要文本对比度 ≥ 4.5:1
- 关键 icon 必须有 `aria-label`(`<view aria-label="返回">`)
- 字体大小可在系统中放大,布局不破

| 服务器错误(5xx) | 我们这边出了点小问题,稍后再试一下吧 |
| 出题超时 | AI 思考超时了,这次不计入次数,要再试一次吗? |
| 出题失败 | 这次没能成功生成,稍后再试一次吧 |
| 内容被拦截 | 内容包含不适宜信息,无法继续,请更换其他素材 |
| OCR 全部失败 | 没能识别到清晰文字,光线不足时可以打开台灯试试 |
| OCR 部分失败 | 这一页有点模糊,可以重拍或跳过 |
| 配额耗尽 | 今日额度已用尽,明日 0 点重置 |
| 上传失败 | 上传不顺,我们重试 3 次都没成功,请检查网络 |
| 体验问题(轻) | 操作过快了,稍等片刻 |
| 答题超时(草稿过期) | 草稿已超过 7 天,我们整理掉了,可以从这里重新开始 |

### 7.3 AI 标识文案(合规)

固定文案,**不可改写**:

> 🤖 内容由 AI 生成,仅供参考

出现要求:
- 出题加载页 footer 永久显示
- 答题页顶部 sticky 半透明显示
- 结果页主区域永久显示
- 单题展开后题旁显示
- AI 主观题批改反馈周围显示

低置信度补充:

> ⚠️ AI 评分置信度较低(< 70%),建议自行判断

### 7.4 空状态文案表

### 7.5 文案语气原则

- **第二人称**:用「你」拉近距离,但不要刻意装可爱
- **少用感叹号**:每屏不超过 1 个;不用!!!
- **避免责备**:错题不是「你又错了」而是「这道题再试一次」
- **诚实**:不夸张「秒级响应」「AI 大脑」;说能做到的就行
