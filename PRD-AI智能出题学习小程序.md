## 一、产品概述

### 1.1 一句话定义

一款基于微信小程序的 AI 学习工具,用户通过**拍照**或从**共享书库**中选择书籍,由 AI 自动出题、批改、解析,帮助用户高效学习任意书籍内容。

### 1.2 核心价值闭环

```
获取书籍内容  →  AI 自动出题  →  在线答题  →  AI 批改并解析  →  错题巩固
```

### 1.3 产品定位

- **类型**:学习辅助工具类小程序
- **目标用户**:通用人群,不限场景(K12 学生、大学生、考研、职业考试、成人自学等均适用)
- **使用场景**:任何需要基于书籍内容自我测试、查漏补缺的学习场景

### 1.4 产品亮点

1. **三种入书方式**:拍照逐页录入、共享书库选书、用户自传 PDF(社区共建)
2. **AI 智能出题**:支持多种题型、难度、题量自定义
3. **AI 智能批改**:不仅判对错,还给出错误原因和改进建议
4. **错题巩固**:自动收录,支持重做
5. **双端合一**:用户端与管理员端在同一小程序中,根据角色动态显示

## 二、用户角色与权限

| 角色 | 权限范围 | 入口与赋权方式 |
|---|---|---|
| **普通用户** | 浏览书库、拍照出题、选书出题、答题、错题本、上传书籍(待审核) | 微信授权登录后默认角色 |
| **管理员** | 普通用户全部权限 + 书籍审核 + 书籍管理 + 用户管理 + 举报处理 | 由超级管理员在后台白名单赋权 |
| **超级管理员** | 管理员全部权限 + 任命管理员 + 系统配置 + 数据大盘 | 项目方自身,数据库直接配置 |

> **重要设计原则(目标形态)**:不做两个独立小程序,在同一小程序中根据用户 `role` 字段动态显示「管理」入口。  
> **MVP 阶段例外**:为压缩开发工作量,V1 (MVP) 阶段管理员使用独立 Web 后台审核与运营,小程序内**不显示**管理入口;V2 起将管理工作台整合进小程序,实现"双端合一"。

## 四、用户端核心流程详解

### 4.1 流程图:从打开到完成一次答题

```
启动小程序
    │
    ▼
微信授权登录(首次需用户确认)
    │
    ▼
首次进入:3 屏引导页;非首次:直接进入首页
    │
    ▼
首页(书库 Tab)
    │
    ├──→ 路径 A:从书库选书
    ├──→ 路径 B:拍照出题(底部 Tab)
    └──→ 路径 C:上传新书(右下浮动按钮)
    │
    ▼
进入「出题配置页」
    │ 选择范围、题型、难度、题量
    ▼
点击「生成试卷」
    │ 后端调用大模型(异步,显示加载动画)
    ▼
进入「答题页」(一题一屏)
    │ 答题、切换、暂存
    ▼
全部答完,点击「提交」
    │ 客观题本地判,主观题 AI 批改(异步)
    ▼
进入「结果报告页」
    │ 总分 + 逐题解析
    ▼
错题自动入库,可选「再来一套」或返回首页
```

### 4.2 三种入书方式对比

| 方式 | 适用场景 | 优点 | 缺点 |
|---|---|---|---|
| **共享书库** | 经典书、教材、热门书 | 即开即用、内容准 | 依赖书库存量 |
| **拍照录入** | 手头纸质书、随时学习 | 灵活、即拍即用 | OCR 有错字、长书慢 |
| **上传 PDF** | 自有电子书 | 一次上传永久使用 | 需等待审核 |

### 4.3 拍照录入的关键 UX 设计

由于这是用户使用频率最高且最容易翻车的环节,必须重点设计:

1. **连拍模式**:点一次拍一张,自动加入待识别列表,无需反复进入相册
2. **缩略图列表**:每张拍完后在底部显示缩略图,可点击预览大图
3. **重拍 / 删除**:每张缩略图右上角小按钮,可单独操作
4. **拖动排序**:支持长按拖动,纠正页面顺序
5. **批量从相册选择**:也支持一次性从相册多选
6. **页数上限**:单次最多 20 页(超出建议直接传 PDF)
7. **OCR 校对**:OCR 完成后,给用户一个**文本预览页**,可编辑修正后再出题(防止错字毁掉出题质量)

#### 4.3.1 性能预期(用户预期管理)

| 阶段 | 单页 | 20 页 | 超出阈值表现 |
|---|---|---|---|
| OCR 单图识别 | ≤ 3 秒 | 整体 ≤ 60 秒(并发 5) | 进度条显示,允许后台运行 |
| 内容安全过审 | ≤ 1 秒/批 | ≤ 5 秒 | 异常时提示「内容审核中,请稍候」 |
| AI 出题生成 | — | ≤ 30 秒 | 超过 30 秒展示「正在思考中,通常 30 秒内完成」 |
| AI 主观题批改 | ≤ 5 秒/题 | — | 整卷批改进度条,逐题点亮 |

### 4.4 出题配置页交互细节

| 配置项 | 选项 | 说明 |
|---|---|---|
| 出题范围 | 整本书 / 选择章节 / 指定页码 | 拍照模式只显示「本次内容」 |
| 题型 | 单选、判断、填空、简答(多选 V2) | 可多选,至少选一个 |
| 难度 | 简单、中等、困难 | 单选 |
| 题量 | 5、10、20、自定义(上限 50) | 单选;新用户首次默认 5 |
| 预估时长 | 系统自动估算 | 给用户心理预期(如:约 8 分钟) |
| AI 内容免责声明 | 固定显示 | 「题目与解析由 AI 生成,仅供学习参考」 |

### 4.5 答题页交互细节

- **一题一屏**(移动端友好,避免长滚动)
- **顶部进度条**:显示「3 / 10」
- **题型适配**:
  - 单选:大按钮单选
  - 多选:多选 + 确认按钮
  - 判断:对 / 错 双按钮
  - 填空:文本输入框
  - 简答:多行文本框,支持暂存
- **底部按钮**:上一题、下一题、暂存退出
- **最后一题**:「下一题」变为「提交」

### 4.6 结果报告页设计

- **顶部数据卡**:总分、正确率、用时、击败 X% 用户
- **AI 标识**:页顶或题旁显眼位置展示「内容由 AI 生成,仅供参考」(合规要求)
- **题目列表**:每题可展开,显示:
  - 题干
  - 我的答案(红色或绿色)
  - 正确答案
  - 错误原因(AI 生成)
  - 涉及知识点
  - 改进建议(主观题)
  - **「申诉」按钮(V2)**:用户对 AI 评分有异议,可一键申诉,进入人工复核队列
- **底部操作**:加入错题本(自动)、再来一套、分享、返回

#### 4.6.1 主观题批改规则

| 项目 | 规则 |
|---|---|
| 评分粒度 | 百分制(每题 0–100,最终按题型权重折算) |
| 模型置信度 | AI 同时输出 confidence(0–1),低于 0.7 在题旁标「⚠️ AI 评分置信度较低,建议自行判断」 |
| 错误原因 | 必须给出具体原因(不少于 20 字),避免「答案错误」之类无效反馈 |
| 改进建议 | 简答题必给,填空题选给 |
| 异议申诉(V2) | 用户提交申诉后 → 人工 24h 内复核 → 复核结果回写并通知 |

#### 4.6.2 分享机制

| 分享形式 | MVP | V2 | 内容 |
|---|---|---|---|
| 小程序卡片(转发好友/群) | ✅ | ✅ | 标题:「我做完了《XXX》的 AI 测验,正确率 X%」,封面:书封面 + 分数 |
| 成绩海报(保存图片) | ❌ | ✅ | 自动生成竖版海报,含书名、分数、二维码,适合朋友圈分享 |
| 邀请奖励 | ❌ | ✅ | 好友通过分享卡片首次进入并完成一次答题,双方各 +5 次出题额度(当天有效) |

### 4.7 异常与降级策略

> 设计原则:用户**任何时候都能拿到一个明确的下一步**,绝不出现"卡住的转圈"或"无字提示"。

#### 4.7.1 出题(AI 调用)异常

| 异常场景 | 表现/降级 | 是否扣次数 |
|---|---|---|
| 大模型超时(> 60 秒) | 自动重试 1 次;仍失败 → Toast「AI 繁忙,请稍后再试」+ 返回配置页 | ❌ 不扣 |
| 大模型返回非法 JSON | 后端再次请求(重试 1 次);两次都失败 → 同上 | ❌ 不扣 |
| 大模型返回题目数量不足 | 取已生成的题目;若 < 50% 目标量 → 视为失败 | ❌ 不扣 |
| 内容安全 API 拦截 | 直接告知「内容含敏感信息,无法出题,请更换素材」 | ❌ 不扣 |
| 大模型主备切换 | 主模型失败时无感切换到备用模型 | 正常扣次 |
| 用户已达每日上限 | 配置页直接灰显「生成」按钮 + 倒计时至零点 | — |

#### 4.7.2 OCR 异常

| 异常场景 | 表现 |
|---|---|
| 图片完全识别不出文字 | 在校对页显示「未识别到文字,请检查光线或重拍」 + 「跳过此页」 |
| 检测到手写/外语 | 提示「检测到手写或外语内容,识别准确率可能较低,请仔细校对」 |
| 单图 OCR 失败 | 该缩略图叠加红色重试图标,可单独重试 |
| OCR 服务整体不可用 | 整批图片排队进入"已拍待处理",后端 30 分钟内重试,完成后用订阅消息通知 |

#### 4.7.3 网络与提交异常

| 异常场景 | 表现 |
|---|---|
| 答题中断网 | 本地缓存当前答题进度;恢复网络后自动同步;最多保留 7 天 |
| 提交时弱网 | 后端使用幂等键(`paper_id + user_id`),重复提交不会创建多份结果 |
| 上传图片失败 | 自动重试 3 次,仍失败提示「网络不佳,请重试」 |
| 拍照页崩溃/退出 | 本地缓存所有未上传的图片(仅本地路径),下次进入恢复列表 |

#### 4.7.4 内容安全拦截的统一文案

- **轻度命中**(色情、广告等):「内容包含敏感信息,无法继续,请更换其他素材」
- **重度命中**(政治、违法等):「检测到违规内容,操作已被记录」+ 后台告警 + 用户次数 -1(防恶意试探)

## 五、管理员端核心流程详解

### 5.1 进入管理后台

管理员登录后,「我的」页底部会出现「进入管理后台」按钮,点击后跳转到管理工作台,**不影响普通用户使用**。

### 5.2 书籍审核流程

```
用户上传 PDF
    │
    ▼
进入待审核队列(数据库 status=0)
    │
    ▼
管理员收到提醒(工作台徽标数)
    │
    ▼
打开待审核列表 → 点击某本书
    │
    ▼
详情页:
  - 封面预览
  - PDF 内嵌预览(用 web-view 嵌 H5 + PDF.js)
  - 上传者信息
  - 用户填写的书籍信息
    │
    ├──→ 点击「通过」→ 入库,通知用户
    ├──→ 点击「驳回」→ 填驳回原因 → 通知用户
    └──→ 点击「编辑后通过」→ 修正字段 → 入库
```

### 5.3 书籍管理流程

- **添加新书**:管理员可绕过审核直接上传(用于官方推荐书)
- **编辑**:修改任何字段
- **下架**:不删除数据,仅 status 改为下架,用户不可见
- **打标签 / 分类**:用于前端筛选和推荐
- **推荐置顶**:出现在首页推荐位

### 5.4 用户管理流程

- **查看记录**:某用户上传过哪些书、答过哪些题
- **封禁**:封禁后用户无法登录,提示「账号异常」
- **管理员任命**(仅超管):将普通用户提升为管理员

## 六、功能优先级与版本规划

### 6.1 MVP(第一版,核心闭环)

> **目标**:验证产品核心价值,4 个月独立开发可完成。

**包含功能**:

## 七、关键产品规则

### 7.1 出题次数限制(防恶意刷 API 成本)

| 用户类型 | 每日出题上限 |
|---|---|
| 新用户(首日) | 5 次 |
| 普通用户 | 10 次 |
| 管理员 | 50 次 |
| 付费用户(V3) | 不限 |

#### 7.1.1 计次细则

- **重置时间**:**每日凌晨 0:00 (Asia/Shanghai)** 重置,前端显示倒计时。
- **计入次数的事件**:成功生成一份试卷(后端返回 `paper_id`)。
- **不计次的事件**:
  - 大模型超时/失败(见 4.7.1)
  - 内容安全 API 拦截
  - 用户在生成过程中主动取消(< 30 秒内)
- **重试不扣**:同一次出题任务的内部重试不重复计数。
- **邀请奖励额度**(V2):当天有效,与基础额度叠加,叠加上限 30 次。

### 7.2 内容安全规则

所有用户产生或 AI 产生的文本,**必须经过微信内容安全 API (`security.msgSecCheck` / `security.imgSecCheck`)**:

| 触发点 | 检测内容 | 拦截后行为 |
|---|---|---|
| 用户拍照(图像) | imgSecCheck | 拒绝上传,提示「图片含违规内容」 |
| OCR 识别后文本 | msgSecCheck | 拒绝出题,提示「内容含敏感信息」 |
| 用户上传的书籍信息(书名/简介) | msgSecCheck | 阻止提交 |
| 用户答题内容(主观题) | msgSecCheck | 答案改为「[内容已屏蔽]」,不影响正常评分 |
| AI 生成的题目和解析 | msgSecCheck | 拒绝展示,自动重新生成一次;仍命中则改用降级模板题 |
| 用户上传的 PDF | msgSecCheck(逐页文本) + imgSecCheck(封面) | 整个 PDF 不入库,通知上传者 |

> **审核日志保留 ≥ 6 个月**(配合"内容审核日志"模块,满足《互联网信息服务管理办法》要求)。

### 7.3 版权与免责

- 用户上传 PDF 必须勾选「我保证拥有版权或为公版书」(默认未勾选,无法提交)
- 服务条款明确「平台仅做技术中转,版权责任由上传者承担」
- 提供版权方投诉下架通道:`copyright@yourdomain.com` + 设置页内表单
- **MVP 阶段共享书库的内容来源**(必须满足以下任一):
  1. 公版书(作者去世满 50 年的中文作品)
  2. 出版社或作者明确授权(留存授权书扫描件)
  3. CC 协议下的开放教材
  4. **不接受**:盗版扫描件、未授权的市售教材

### 7.4 数据保留与注销

| 数据类型 | 保留时长 |
|---|---|
| 用户账号信息 | 至用户主动注销时清除 |
| 拍照临时内容 | 7 天后自动清理 |
| OCR 文本临时内容 | 出题完成后保留 30 天 |
| 答题记录 | 用户在线期间永久,注销时一并删除 |
| 错题本 | 同上 |
| 上传的 PDF(已通过审核) | 永久(用户注销后转为匿名公共书,保留可访问) |
| 上传的 PDF(被驳回) | 30 天后清理 |
| 内容安全审核日志 | ≥ 6 个月(法律要求,加密留存) |
| 客服/反馈记录 | 1 年 |

#### 7.4.1 用户注销流程(满足《个人信息保护法》)

1. 「设置 → 账号与安全 → 注销账号」
2. 7 天冷静期(期间登录则取消注销)
3. 注销生效后:**24 小时内**完成上述「需删除」类数据清理
4. 邮件/订阅消息通知确认

### 7.5 法律与合规

#### 7.5.1 用户协议与隐私政策

- **首次启动小程序**弹出协议确认弹窗(《用户协议》《隐私政策》),不勾选无法使用
- **重大变更**:协议更新后再次弹出,要求重新同意
- **隐私政策必须明确披露**:
  - 收集的信息(微信昵称/头像 OpenID/拍照内容/答题数据)
  - 使用目的(提供服务/出题批改/错题分析)
  - 第三方共享(云厂商存储/大模型服务商,需列明具体名称)
  - 用户权利(查看/更正/删除/注销/投诉)

#### 7.5.2 未成年人保护(目标用户含 K12)

- 「设置」中提供「**未成年人模式**」开关(由家长协助开启)
- 未成年人模式特点:
  - 每日累计使用时间上限 60 分钟,超时强制退出
  - 23:00 – 06:00 禁用
  - 不开放分享、社区(V3)等社交功能
  - 出题次数减半
- 注册时若未授权头像昵称,默认按 14 岁以上对待;一旦用户在资料中标记未成年,自动启用该模式
- 设置「儿童个人信息保护规则」专门页面

#### 7.5.3 生成式 AI 合规(《生成式 AI 服务管理暂行办法》)

- 所有 AI 生成的内容(题目、解析、批改)必须**显眼标识**「内容由 AI 生成,仅供参考」
- 必须使用**已通过备案**的国内大模型(参考国家网信办公布的算法备案清单),如:
  - 主选:DeepSeek (深度求索) — 已备案
  - 备选:百度文心一言、阿里通义千问、智谱 GLM 等
- 禁止使用未备案的境外模型(如 OpenAI、Anthropic)
- 后台保留所有 AI 调用日志(prompt + response)≥ 6 个月

#### 7.5.4 备案与资质

| 项目 | 要求 | 时间 |
|---|---|---|
| ICP 备案 | 域名备案 | 提前 20 工作日 |
| 公安联网备案 | ICP 通过后 30 日内 | 取得 ICP 后 |
| 小程序类目 | 选「工具 - 学习记录工具」 | 注册时确认 |
| 生成式 AI 算法备案 | 使用已备案的大模型即可,无需自行备案;但需在「关于」页公示所用模型 | MVP 上线前 |

### 7.6 错题与学习规则

#### 7.6.1 错题本判定

- **自动入库**:答题结束时,所有非满分题目(客观题错;主观题 < 60 分)自动加入错题本
- **去重规则**:同一题(题干 hash 相同)只保留最新一次错误记录
- **「掌握」标准**(满足任一即可):
  - 用户手动标记「已掌握」
  - 在错题重做模式下,**连续答对 2 次**(自动)
- **掌握后**:从默认重做列表中隐藏,但可在「已掌握」标签页查看;若 30 天后再次答错则重新加入

#### 7.6.2 暂存退出

- 答题进度本地缓存 + 后端同步
- 保留期限 **7 天**(超时清理,防止试卷题目过时)
- 跨设备同步:同账号在新设备登录可恢复

### 7.7 消息通知机制

- **形式**:微信「订阅消息」(模板消息已废弃)
- **订阅时机**:用户首次触发通知场景前弹出订阅授权(单次订阅)
- **MVP 通知场景**:暂无(MVP 不含上传/审核流)
- **V2 起通知场景**:
  | 场景 | 模板 | 是否一次性 |
  |---|---|---|
  | 书籍审核通过 | 书名/审核人/通过时间/感谢语 | 一次性 |
  | 书籍审核驳回 | 书名/驳回原因/重试入口 | 一次性 |
  | 错题打卡提醒(V3) | 错题数/连续天数/打卡链接 | 长期(每日) |
  | 申诉处理结果 | 申诉时间/复核结论/复核分数 | 一次性 |

## 九、产品风险与应对

| 风险 | 应对方案 |
|---|---|
| **OCR 错字影响出题质量** | 强制「OCR 后编辑校对」步骤,用户确认后再出题 |
| **大模型 API 成本失控** | 出题次数限制 + 客观题本地批改 + 选用便宜模型(如 DeepSeek) + 主备模型机制 |
| **大模型不稳定/超时** | 主备模型自动切换;失败重试不计次;前端 60 秒超时友好提示 |
| **大模型输出不规范 JSON** | Prompt 工程 + JSON Schema 校验 + 失败重试 + 降级模板题 |
| **版权投诉** | 用户协议免责 + 审核机制 + 投诉下架通道 + 严格的书库来源管控(7.3) |
| **微信类目审核不通过** | 选「工具」类目,定位「学习记录工具」,避免「教育」「考试」字眼 |
| **生成式 AI 备案要求** | 接入已备案的国内大模型 + 明确 AI 内容免责声明 + 在「关于」页公示模型 |
| **小程序备案周期** | 提前 20+ 工作日办理 ICP 与公安备案;先开始备案再开发 |
| **PDF 内嵌预览受限** | web-view 须使用备案 HTTPS 域名;在小程序后台「业务域名」白名单配置;V2 实现前先验证可行性 |
| **未成年人保护合规** | 设置-未成年人模式 + 时长/夜间限制 + 关闭社交;注册时探测年龄 |
| **个人信息保护合规** | 完整隐私政策 + 用户注销流程 + 24 小时内删除数据 |
| **拍照体验差** | 重点投入连拍 / 缩略图 / 重拍 / 排序 UX 设计 |
| **冷启动书库内容稀薄** | MVP 上线前管理员预先上传 50+ 公版书 / 教材 |
| **OCR 第三方服务故障** | 双供应商容灾(微信 OCR + 腾讯云 OCR / 百度 OCR);异步队列重试 |
| **AI 评分用户不认可** | V2 引入申诉机制 + 人工复核;主观题在结果页明示「AI 评分仅供参考」 |
